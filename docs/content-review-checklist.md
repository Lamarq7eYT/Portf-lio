# Portfolio Content Review Checklist

Use this checklist before updating project cards, case studies, or deployment
links in the portfolio.

## Project Copy

- State the problem solved by the project.
- Mention the strongest technical decision.
- Keep stack labels current.
- Link to the repository or live demo when available.
- Avoid vague claims that are not visible in the project.

## Visuals

- Check desktop and mobile screenshots.
- Confirm images have useful alt text.
- Make sure animations do not hide core content.
- Keep the first viewport focused on the actual work.

## Release Check

- Run the local build.
- Open the deployed URL after publishing.
- Test contact links and project links.
- Note any unfinished work as roadmap instead of pretending it is complete.
