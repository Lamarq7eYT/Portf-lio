# Flagship Project Case Study Guide

The portfolio can communicate impact better when each flagship project has a compact case study instead of only a card or repository link.

## Recommended Structure

Each case study should include:

1. Problem: what the project is trying to solve or demonstrate.
2. Approach: architecture, design direction, and trade-offs.
3. Stack: main languages, frameworks, and tools.
4. Result: what works today and what the project proves.
5. Link: repository, demo, screenshots, or video where available.

## Initial Projects

| Project | Angle |
| --- | --- |
| AegisHub | Security scanner, CLI workflow, dashboard UX, Rust + TypeScript integration. |
| AegisCore | Defensive web platform, auth/rate-limit posture, admin demo, observability. |
| HackerPage | Interactive educational security experience with visual storytelling. |
| Polybech | Polyglot benchmark lab and comparison dashboard. |

## Copy Rules

- Keep each case study short and scannable.
- Prefer concrete details over generic claims.
- Include one technical challenge and one design decision per project.
